## 🎯 الجهة والمشروع

**الجهة:** جمعية المسؤولية الاجتماعية
**المشروع:** نظام إدارة مستودعات تشغيلية (عُدّة المبادرات وليس مواد غذائية)
**المستخدم:** غير مبرمج — يحتاج شرحاً مفصّلاً بالعربية لكل خطوة.

---

## 🏢 البنية الفيزيائية للمستودع

| العنصر | المواصفة |
|:---:|:---:|
| **مساحة المستودع** | 4 متر × 4 متر |
| **عدد مساحات التخزين** | 4 (سُمّيت A، B، C، D) |
| **توزيع المساحات** | A و B على الجدار الأيمن، C و D على الجدار الأيسر، ممرّ في المنتصف |
| **أبعاد كل مساحة** | عرض 200سم × ارتفاع 230سم × عمق 65سم |
| **الأرفف داخل كل مساحة** | 3 أرفف أفقية |
| **الصناديق على كل رف** | 3 إلى 4 صناديق |

### نظام الترميز (مهم جداً)
```
[المساحة]-[الرف]-[الصندوق]
مثال: A-2-3 = المساحة A، الرف الثاني، الصندوق الثالث
```

### تصنيف المساحات الافتراضي
- **A:** عُدّة الفعاليات الميدانية
- **B:** العُدّة التقنية والإلكترونية
- **C:** تجهيزات الإقامة الميدانية
- **D:** المواد التشغيلية المساندة

## ✅ ما تمّ إنجازه (في محادثة claude.ai السابقة)

### 1. تصميم النظام الكامل
- نموذج أوّلي تفاعلي تمّ اعتماده من المستخدم.
- تحديد جميع الميزات (42 وظيفة موزّعة على 8 مجموعات).
- اعتماد البنية البصرية (المنظور العلوي + المنظور الداخلي للمساحات).

### 2. بناء التطبيق (موجود في هذا المجلد)
- **35 ملفاً** و**2461 سطر كود**.
- React 18 + Vite + Tailwind CSS.
- Supabase للمصادقة وقاعدة البيانات.
- 12 مكوّن React + 3 صفحات + AuthContext كامل.
- سكربت SQL لإعداد قاعدة البيانات.

### 3. الميزات المُنفّذة
- ✅ نظام مصادقة كامل (تسجيل دخول/تسجيل/طلبات انضمام).
- ✅ منظور علوي للمستودع بالأبعاد الواقعية.
- ✅ منظور أمامي للمساحة (200×230سم) مع الأرفف والصناديق.
- ✅ بحث عن غرض مع تمييز موقعه.
- ✅ نظام إخراج بمهلة افتراضية **10 أيام** + خيار "غرض شخصي".
- ✅ تنبيه شديد أحمر عند تجاوز المهلة (يومض).
- ✅ ثلاث آليات لمعالجة المُخرَج: **إرجاع / إتلاف / دعم**.
- ✅ سجل المتلفات مع إمكانية الاسترجاع.
- ✅ سجل الدعم (الجهة + المبادرة).
- ✅ نظام صلاحيات دقيق: 6 صلاحيات مستقلة (عرض/إخراج/إرجاع/إضافة/تعديل/حذف).
- ✅ ثلاثة أدوار: مدير نظام، مدير مستودع، مستخدم.
- ✅ موافقة مدير المستودع على طلبات الانضمام مع تحديد الصلاحيات.
- ✅ سجل تتبّع كامل لكل العمليات.
- ✅ تقارير تحليلية (الأكثر استخداماً، إحصائيات).
- ✅ توليد رموز QR للستيكرات الخمسة (مدخل + 4 مساحات).
- ✅ اقتراح ذكي لموقع تخزين الأدوات الجديدة.

## ⏳ ما لم يُنجَز بعد (المهمّة الحالية)

### المرحلة 1: اختبار التطبيق محلياً
- تثبيت `npm install`.
- تشغيل `npm run dev`.
- التأكّد من عمل كل شيء قبل النشر.

### المرحلة 2: إعداد Supabase
1. مساعدة المستخدم في إنشاء حساب على https://supabase.com
2. إنشاء مشروع جديد.
3. تنفيذ سكربت `supabase/setup.sql` في SQL Editor.
4. الحصول على `Project URL` و`anon public key`.
5. إضافتها لملف `.env`.

### المرحلة 3: رفع الكود لـ GitHub
1. إنشاء حساب GitHub (إن لم يكن موجوداً).
2. إنشاء مستودع جديد `sra-warehouse`.
3. رفع المشروع باستخدام `git`.

### المرحلة 4: النشر على Vercel
1. إنشاء حساب Vercel وربطه بـ GitHub.
2. استيراد المشروع.
3. إضافة متغيّرات البيئة (`VITE_SUPABASE_URL` و`VITE_SUPABASE_ANON_KEY`).
4. النشر.

### المرحلة 5: إعداد الحساب الأول
1. إنشاء مستخدم في Supabase Auth.
2. تنفيذ SQL لجعله `sysadmin` ومدير المستودع الرئيسي.
3. التحقّق من كل شيء يعمل.

## 🔑 معلومات مهمة لـ Claude Code

### المستخدم لا يبرمج
- اشرح كل أمر `bash` تنفّذه ولماذا.
- لا تستخدم مصطلحات تقنية معقّدة بدون شرح.
- استخدم العربية الفصحى الرسمية.
- اعرض الخيارات بدلاً من اتخاذ القرارات نيابةً عنه.

### تفضيلات المستخدم
- يحب الجداول والقوائم المنظّمة.
- يفضّل الردود الموجزة المباشرة.
- يحتاج تأكيد قبل الخطوات المهمّة.

### نقاط حسّاسة
- ⚠️ **لا تنشئ حسابات نيابةً عنه** — هذه تتطلّب بريده الإلكتروني.
- ⚠️ **لا تطلب كلمات مرور** أبداً.
- ✅ مفاتيح API (anon key) آمنة للمشاركة.

## 🎯 الخطة المُقترحة لـ Claude Code

1. **تأكيد سلامة المشروع** (`npm install` ثم `npm run dev`).

6. **اختبار شامل** للتأكّد من كل شيء.
7. **توليد رموز QR** للطباعة على الستيكرات.
